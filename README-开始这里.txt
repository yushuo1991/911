═══════════════════════════════════════════════════════════
🎯 两种自动化部署方案
═══════════════════════════════════════════════════════════

【方案 1：服务器定时检查】（推荐 - 真正的完全自动化）
  ✅ 完全无需本地操作
  ✅ 服务器自动检查 GitHub 更新
  ✅ 自动拉取代码并部署
  ✅ 只需在服务器配置一次 cron 定时任务

  快速开始：
    1. 双击运行 "GitHub-CLI自动化.bat"
    2. 按提示完成 GitHub 仓库创建
    3. 在服务器上运行 server-deploy-setup.sh
    4. 配置 cron 定时任务（详见 方案1-执行指南.txt）
    5. 完成！之后只需本地 git push，服务器自动部署

【方案 2：GitHub CLI 自动部署】（本地推送触发）
  ✅ 使用 GitHub CLI 简化操作
  ✅ 一个命令完成推送和部署
  ✅ 实时查看部署进度

  快速开始：
    - Windows：双击 "GitHub-CLI自动化.bat"
    - 或手动运行：powershell -ExecutionPolicy Bypass -File auto-deploy-github.ps1

💡 推荐开始方式
═══════════════════════════════════════════════════════════

1. 【最快】双击 "GitHub-CLI自动化.bat"
   - 自动安装所需工具
   - 引导完成所有配置
   - 推送代码到 GitHub

2. 【完全自动化】配置服务器定时检查
   - 阅读 "方案1-执行指南.txt"
   - 在服务器运行初始化脚本
   - 配置 cron 定时任务

3. 完成！之后每次修改代码：
   - git add .
   - git commit -m "更新说明"
   - git push
   - 服务器自动检测并部署 ✨
